class oak_deepseek.engine.AgentEngine[源代码]¶

基类：object

核心类，用于创建和运行Agent系统。 引擎本身是无状态的，只保存注册信息。实际运行状态由AgentCore管理， 支持断点重连：通过create_core传入历史消息列表即可恢复之前的执行状态。

create_agent(key: Tuple[str, str], description: str, prompt: str, tools: List[Callable] | None = None, with_stream: bool = False, sub_agents: List[Tuple[str, str]] | None = None)[源代码]¶

注册一个Agent至引擎，以后可以用唯一命名空间+名字来指定。

参数:

  • key -- 命名空间+名字，例如 ("sys", "math_agent")

  • description -- Agent的简短描述

  • prompt -- Agent的系统提示词

  • tools -- 可选，Agent可调用的工具函数列表

  • sub_agents -- 可选，可调用的子Agent列表，每个元素为子Agent的key

返回:

None

create_core(key: Tuple[str, str] | List[Tuple[Tuple[Tuple[str, str], ...], Annotated[SystemMessage | UserMessage | AssistantMessage | ToolMessage, FieldInfo(annotation=NoneType, required=True, discriminator='role')]]], history_queue: Queue, raw_response_queue: Queue[RequestResponsePair] | None = None, api_key: str | None = None) → AgentCore[源代码]¶

初始化引擎，指定入口Agent和消息记录队列。

支持两种模式：

  • 正常启动：传入一个元组 (namespace, name) 作为入口Agent的key。

  • 断点恢复：传入一个历史消息列表，每个元素为 (key_chain, message)，按时间顺序排列。 引擎会根据消息中的key_chain重建调用栈和各Agent的消息历史。

参数:

key -- 启动方式。

  • 若为元组 (namespace, name)，表示正常启动，该元组为入口 Agent 的 key。

  • 若为列表，表示断点恢复模式。列表元素为 (key_chain, message)，按时间顺序排列。传入空列表时，会抛出 ValueError。

  • 注意：恢复模式下传入的历史消息列表可能会被修改（恢复时会补全缺失消息以确保符合API规范）

参数:

  • history_queue -- 消息输出队列，运行期间产生的所有消息（附带调用链信息）都会被放入此队列

  • raw_response_queue -- 可选，用于输出原始请求/响应对的队列

  • api_key -- DeepSeek API密钥，默认从环境变量DEEPSEEK_API_KEY读取

返回:

已初始化的AgentCore实例

run(input_queue: Queue[str], key: Tuple[str, str] | List[Tuple[Tuple[Tuple[str, str], ...], Annotated[SystemMessage | UserMessage | AssistantMessage | ToolMessage, FieldInfo(annotation=NoneType, required=True, discriminator='role')]]], history_queue: Queue, raw_response_queue: Queue[RequestResponsePair] | None = None, api_key: str | None = None)[源代码]¶

开始任务，阻塞直到所有Agent完成工作。 此方法会：

  1. 调用 create_core 创建 AgentCore 实例（支持正常启动或历史恢复）。

  2. 正常启动时，从 input_queue 获取初始任务；恢复时，直接从历史中恢复执行状态，不再读取初始任务。恢复时，传入循环函数的 task 参数为 None，会被 init 函数自动忽略。

  3. 反复调用 main 函数处理消息循环，直到调用栈为空

注意：在恢复模式下，input_queue 仍然用于在 ReactiveReAct 循环中接收新的用户输入。

参数:

  • input_queue -- 用户输入队列，用于接收新消息（在ReactiveReAct模式下需要）

  • key -- 同create_core的key参数，用于确定启动方式

  • history_queue -- 消息输出队列，同create_core

  • raw_response_queue -- 可选，原始请求/响应队列

  • api_key -- DeepSeek API密钥

返回:

None

class oak_deepseek.core.AgentCore(agent: Agent, history_queue: Queue[Tuple[Tuple[Tuple[str, str], ...], Annotated[SystemMessage | UserMessage | AssistantMessage | ToolMessage, FieldInfo(annotation=NoneType, required=True, discriminator='role')]]], api_key: str | None = None, raw_response_queue: Queue[RequestResponsePair] | None = None)[源代码]¶

基类：object

Agent运行时的核心，管理当前Agent、调用栈和消息历史。

变量:

  • history_queue -- 消息输出队列，每条消息会附带其调用链

  • client -- ChatClient实例

  • agent -- 当前Agent实例

  • stack -- 调用栈，存储祖先Agent实例

  • memory -- 按 key_chain 索引的消息历史缓存，用于 subagent 的有状态记忆

back()[源代码]¶

返回到父Agent：将当前 Agent 的消息历史保存到 memory 中，然后弹出栈顶，恢复父Agent为当前Agent。

send() → AssistantMessage[源代码]¶

基于当前Agent的消息历史调用DeepSeek API，返回助手消息。 该消息会自动添加到当前Agent的消息列表，并放入历史队列。

返回:

助手消息

sub_agent(agent: Agent)[源代码]¶

切换到子Agent：深拷贝当前Agent入栈，并将当前Agent设为子Agent。 如果 memory 中存在该子 Agent 的 key_chain 对应的历史消息，则直接替换子 Agent 的消息列表（不拼接），以实现有状态记忆。

参数:

agent -- 子Agent实例

class oak_deepseek.agent.Agent(key_chain: Tuple[Tuple[str, str], ...], info: AgentInfo)[源代码]¶

基类：object

Agent实例，包含其元数据和消息历史。

变量:

  • key_chain -- 调用链元组，每个元素是 (namespace, name)，表示从根 Agent 到当前 Agent 的路径。 例如 (('sys','root'), ('sys','calculator'))。

  • info -- AgentInfo对象，包含静态配置

  • messages -- 该 Agent 的消息历史列表，按时间顺序排列。 消息会被 AgentCore 持久化到队列，并在恢复时重新加载。

class oak_deepseek.agent.AgentFactory[源代码]¶

基类：object

Agent工厂，负责注册和构建Agent实例。

主要功能： - 通过 register_agent 注册静态配置（AgentInfo）。 - 通过 build 根据 key_chain 动态创建 Agent 实例。 - 如果 Agent 配置了子 Agent（sub_agents 非空），则自动：

  1. 将 choose_agent 工具加入工具列表。

  2. 将子 Agent 的描述信息（命名空间、名字、简介）拼接到提示词中。

build(key_chain: Tuple[Tuple[str, str], ...]) → Agent[源代码]¶

根据 key_chain 构建一个 Agent 实例。

构建步骤： 1. 验证 key_chain[-1] 是否已注册，否则抛出 KeyError。 2. 深拷贝对应的 AgentInfo，防止修改原始配置。 3. 创建 Agent 实例，初始消息列表为空。 4. 如果 sub_agents 非空：

  • 将 choose_agent 工具添加到工具列表。

  • 遍历 sub_agents，验证每个子 Agent 已注册，并将它们的描述信息拼接到提示词末尾。

参数:

key_chain -- 调用链元组，最后一个元素是要构建的 Agent 的 key。

返回:

构建好的 Agent 实例（消息列表为空）

抛出:

KeyError -- 如果 key_chain[-1] 或任何 sub_agent 未注册

register_agent(key: Tuple[str, str], agent: AgentInfo)[源代码]¶
